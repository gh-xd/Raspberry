4.1 安装Virtualenv

4.1 安装Virtualenv

为什么要安装Virtualenv

一般会使用Conda（Anoconda）来创建虚拟环境。然而，树莓派是ARM架构，Conda仅支持x86架构，buster镜像是32位的系统。

另外，树莓派上的Conda代替方案是Miniconda，但是许久不更新，且坑多。

所以，在树莓派中使用Virtualenvl作为管理虚拟环境。

操作过程

采用apt-get的安装方式

sudo apt-get install virtualenv

才用其他方式？

其它

查看树莓派里通过apt-get安装过的包

apt list --installed

结果太多，用grep进行过滤

apt list --installed | grep
